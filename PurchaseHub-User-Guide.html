1. What is PurchaseHub?

PurchaseHub is a simple web tool for requesting and tracking purchases inside the organisation. Employees raise purchase requests; an admin reviews them, places the order, records the payment and delivery status, attaches the invoice, and marks each item as delivered. Everyone sees the same live list, so there is no need to chase updates over chat or email.

The screen updates automatically every few seconds, so any change an admin makes appears for everyone without refreshing the page.

2. Roles & access

There are two types of users:

RoleCan doCannot do
Employee Create new purchase orders; view all orders; add notes to any order. Change status, payment, tracking link, delivery date, invoice; delete orders.
Admin Everything an employee can do plus update status, payment, tracking, delivery date, upload invoices, and archive (delete) orders. —
Keep credentials private. Accounts are shared per role. Do not post the password in public chats. Contact your administrator if you need access.

3. Signing in

  1. Open https://purchase.psitech.co.in in your browser.
  2. Enter your Username and Password.
  3. Use the eye icon in the password box to show or hide what you typed.
  4. Click Sign In.

If the username or password is wrong, a red message appears. If you see “Cannot reach server,” the application may be temporarily down — try again shortly or contact your administrator.

4. The dashboard at a glance

After signing in you land on the dashboard. The top of the screen shows four summary cards:

CardMeaning
New OrdersTotal number of active orders in the system.
Pending OrdersOrders raised but not yet being processed.
In Process OrdersOrders the admin has started working on / ordered.
Delivered OrdersOrders that have arrived.

Click any card to jump straight to the order list filtered by that status. Below the cards is the New Purchase Order form.

5. Creating a new purchase order

Anyone (employee or admin) can raise a request. On the dashboard, fill in the New Purchase Order form:

FieldRequired?What to enter
Requester NameYesYour name (who needs the item).
Project NameYesThe project or department the purchase is for.
Product NameYesWhat you want to buy.
Product URLOptionalA link to the product page, if you have one.
QuantityYesHow many units (minimum 1).
Product DescriptionOptionalSpecs, colour, size, or any notes for the buyer.
Needed By DateYesThe date you need it delivered by.

Click Submit Order. A green “Order created successfully.” message confirms it was saved, and the form clears for the next entry. New orders always start as Pending and Unpaid.

Tip: You don’t need to refresh — the new order appears in the Purchase Orders list automatically.

6. Viewing & tracking orders

Click Purchase Orders in the top bar (or any dashboard card) to open the full list. Each row shows the product, quantity, project, requester, order date, the date it is needed by, current status, payment status, delivery date, invoice, and notes.

Filtering the list

Use the summary buttons above the table to filter:

  • Total — all active orders.
  • Pending / In Process / Delivered — only orders in that status.
  • Archive — orders that have been deleted/archived (kept for reference).

Adding a note (everyone)

Any user can type into the Notes box on a row and click the Save (disk) icon in the Action column. Use notes for comments like “urgent,” “approved by manager,” or vendor remarks.

7. Admin: updating an order Admin only

When signed in as Admin, the table rows become editable. For each order you can:

  • Tracking URL — paste the courier/tracking link. A small open-link icon appears so you can open it in one click.
  • Order Status — choose Pending, In Process, or Delivered.
  • Payment — set Unpaid or Paid.
  • Delivery Date — pick the date the item arrived.
  • Invoice — upload the bill (see next section).
  • Notes — add internal remarks.
Important: After changing anything in a row, click the Save (disk) icon in the Action column to store it. A confirmation message appears. Unsaved edits are not kept.

8. Invoices

Admin To attach a bill to an order:

  1. In the order’s Invoice cell, click Upload.
  2. Choose a file (PDF, Word document, or an image).
  3. Click the Save icon on that row to store it.

Once uploaded, the file name appears as a chip. Any user can click that chip to open and view the invoice. Use the small upload icon next to it to replace the file.

9. Archiving (deleting) orders Admin only

To remove an order from the active list, click the trash icon in its Action column. A confirmation box appears — click Move to Archive. The order is not permanently destroyed; it moves to the Archive filter where it can still be viewed.

10. Switching light / dark theme

Click the moon / sun icon in the top bar to switch between light and dark appearance. Your choice is remembered on your device for next time.

11. Signing out

Click the logout icon (the arrow at the far right of the top bar) to sign out. You will be returned to the login screen. Always sign out on shared computers.

12. Frequently asked questions

I clicked Submit but got “Failed to create order.”

This usually means your session has expired. Sign out and sign back in, then try again. If it continues, contact your administrator.

I made a change as admin but it didn’t stick.

You must click the Save (disk) icon on that row after editing. Edits are not saved automatically.

Someone else placed an order — why don’t I see it?

You should: the list refreshes every few seconds. If not, reload the page or check your internet connection.

Can an employee change the status or delete an order?

No. Employees can create orders and add notes only. Status, payment, tracking, invoices, and deletion are admin actions.

Where do deleted orders go?

They move to the Archive filter and remain viewable; they are not permanently erased.
